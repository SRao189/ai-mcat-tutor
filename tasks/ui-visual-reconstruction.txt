Known repo reality (read this before building — don't re-derive it):

- No JS framework, no bundler. Plain HTML/CSS/vanilla JS + a Python stdlib
  server. Keep it that way — don't introduce a framework or build step for
  three screens.
- There are currently TWO "homepage" surfaces: `server/app.py::_serve_index()`
  (a bare `<ul>` of raw section IDs — this is the literal "developer index"
  being replaced) and `app/index.html` (already a more built-out shell with
  sidebar nav and localStorage progress, but only covers 5 of 11 modules).
  Recommended: extend `app/index.html` to cover all 11 modules and make it
  the canonical root; retire the bare list. If you find a reason to keep both
  surfaces, say so explicitly instead of leaving two inconsistent chapter
  lists live.
- The chapter/module list is hardcoded in two places with no shared registry
  (`council/course_modules.py::MODULE_BY_SECTION` and `app/index.html`'s
  `COURSE_UNITS`). For these three screens, read directly from
  `course-data/*.json` — building a full registry/metadata system is a
  separate, later task, not this one.
- "Open Notebook" is not integrated anywhere in this repo (only an unpulled
  git remote). Don't reference it as a working system in UI copy, comments,
  or commit messages.
- NVIDIA Council (`council/`) is real and working — the citation-gated tutor
  backend. Don't modify its logic. The Lesson page rebuild only changes how
  its answers are displayed, not how they're produced.

Hard constraints — none of the following may appear in the shipped UI:

- No gradients
- No giant hero heading
- No raw section IDs anywhere visible (thermo, chapter-3-summary,
  complex-ii-sdhd, etc.) — clean learner-facing names only
- No random icon library — pick one small consistent approach, or ship with
  no icons at all
- No dashboard full of meaningless cards
- No fake streaks or metrics without real backing data — every number shown
  must come from actual course-data/progress state, never invented
- No chatbot permanently occupying half the page — tutor panel is closed by
  default on the Lesson page and opens on request
- No decorative AI imagery
- No excessive rounded rectangles
