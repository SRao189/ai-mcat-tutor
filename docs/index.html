Flagship case study

From source packet to student-facing lesson.

The point of this project is not simply generating study text. It is generating lessons while preserving explicit checks for what can be trusted, what must be rejected, and what needs review before shipping.

What the learner sees

Structured chapter lessons with explanations, worked examples, and practice prompts that feel like a study product rather than a raw model output.

  • Five public module previews
  • Worked examples and practice questions
  • Lesson checks and chapter progression

What the system checks

The generation pipeline enforces artifact integrity, citation integrity, and contradiction-safe claim support before lesson content is considered ready to ship.

  • Gate 1: artifact integrity
  • Gate 2: citation integrity
  • Gate 3: claim-support validation

What is public here

This deployment excludes source PDFs, environment files, logs, and internal research artifacts. The public site only ships the static assets and preview data needed for presentation.

  • 56 repo test definitions counted from current test files
  • Benchmark summary values copied into the portfolio narrative
  • No local filesystem paths or hidden inputs exposed
